description: Write a conventional-commits title and PR description for the current branch

Use the `pr-description` skill to write a PR title and body for the current branch against its base. Follow the skill's four-step process exactly: gather git context, separate known from unknown, ask clarifying questions, then write.

Do not skip the clarifying-questions step, even on small diffs. Do not open the PR, push, or mutate git state.
